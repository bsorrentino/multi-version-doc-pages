# Multi-Version Documentation Portal

Welcome to the Multi-Version Documentation Portal. This page is a proof of concept demonstrating how to deploy multiple versions of documentation to GitHub Pages using `mkdocs`, `mkdocs-material`, and `mike`.

## Features

- **Multi-Version Support**: Seamlessly switch between different versions of documentation via the header drop-down selector.
- **Git Flow Integration**: Automatic publication of updates from the `develop` branch to the `dev` version, and git tags (e.g., `v1.0.0`) to specific version subdirectories.
- **Custom Portal Landing Page**: A beautifully designed root portal page that lists all available versions.

## How to Navigate

Use the version selector located in the header of the documentation to transition between versions:

- **dev**: The development version, representing the latest changes in the `develop` branch.
- **Releases (e.g., 1.0.0)**: Stable releases mapped to git tags.
